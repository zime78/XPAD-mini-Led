# 프로파일 단축 전환 GUI 시안

## 상태

- 단계: 구현 및 실기기 확인 완료
- 기준 화면: 사용자 제공 재생 창 캡처와 현재 `PlayerView`·`PlayerStatus`·`styles.css`
- 시안: [`profile-quick-switch-gui.svg`](./profile-quick-switch-gui.svg)
- 구현 화면: [`profile-quick-switch-implemented.png`](./profile-quick-switch-implemented.png)

## 권장 구성

1. USB·LCD·노브 상태 아이콘 오른쪽에 구분선과 간격을 둔다.
2. 별도 `QUICK PROFILE` 문구 없이 `P1`~`P5` 단축 버튼만 배치한다.
3. 실제 사용 중인 프로파일은 파란 배경·밝은 테두리·상단 점으로 색 이외의 상태 단서도 제공한다.
4. 프로파일 버튼 바로 아래에는 별도 제목이나 바깥 묶음 없이 왼쪽·가운데·오른쪽 버튼 3개만 독립적으로 표시한다.
5. 각 버튼은 슬롯명과 실제 선택값을 분리하고, 선택값은 파란 배경·테두리의 내부 캡슐로 강조한다.
6. 키 이름이 길면 한 줄 말줄임표와 전체 이름 툴팁을 사용하고, 프로파일 전환 중에는 다섯 버튼을 잠시 비활성화한다.

## 시안에서 사용한 예시 데이터

- 사용 중 프로파일: `P3`
- 왼쪽 버튼: `Q`
- 가운데 버튼: `Safari` 실행
- 오른쪽 버튼: `F5`

표시값은 고정 문구가 아니라 현재 저장된 프로파일 설정에서 가져오는 동적 데이터다. `P1`을
선택하면 고정값인 `이전 곡`·`재생/일시정지`·`다음 곡`을 표시한다.

## 구현 경계

- 상단 버튼은 편집 탭 선택이 아니라 실제 XPAD `cfg_selection` RAM 프로파일 전환을 의미한다.
- 전환 성공 표시는 SystemInfo readback으로 선택값을 확인한 뒤에만 갱신한다.
- 연결 끊김·프로토콜 미준비·전환 실패 때는 버튼을 비활성화하고 기존 프로파일 표시를 유지한다.
- 장치 플래시 저장과 `Save(0x0D)`는 사용하지 않는다.
